## Key Decisions

### 1. Re-sign After Tauri Build Rather Than During
Tauri does not support hardened runtime code signing natively in its config. Rather than patching the Tauri build process, the `.app` was extracted from the DMG, re-signed externally with the correct flags, and the DMG was rebuilt. This keeps the Tauri build clean and the signing step explicit and auditable.

### 2. Candidate List in copy-dist.js Instead of a Single Path
Using a priority-ordered candidate list allows the same script to work for both OpenSilver builds (old path) and Blazor WASM builds (new path) without requiring a flag or env var. This avoids breaking other developers or CI configurations that may use either toolchain.
